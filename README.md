These codes run the GFlowNet extension used in the Smolyak Hierarchy-Inspired Protocol (SHIP). The main code is main_gfn_extend.py, with potts.yaml being the config file. Several variables are assigned at runtime at the start of the main code. In particular, pathsum_input_file, dynamics_executable, and starting_paths_input_file1 are assigned in the sbatch submission script, submit_gfn_pathsum9. Changing these variables allows multiple instances of the GFlowNet code to be run in parallel, from different starting points. The modified Fortran codes are for the quantum dynamics calculations. The file specific-paths-len3-inds1-solution184paths-reward149.42207.inp contains the starting set of spin pathways, which are read as input into the GFlowNet code. The generated spins from the GFlowNet are then appended to the corresponding pathways, and then the larger pathways are sent to the Fortran codes to compute the quantum dynamics, which are sent back to the GFlowNet to compute the dynamics-based reward out of 150. The GFlowNet code writes the top-scoring set of pathways from each validation step to an output file as it runs, to be stored for analysis.
